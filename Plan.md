## Перечень используемых инструментов с обоснованием выбора
* **Программа IntelliJ IDEA** - интегрированная среда разработки (комплекс программных средств, который используется для написания, исполнения, отладки и оптимизации кода.В области тестирования кода среда также может предложить инструменты для проведения автоматических тестов и формирования аналитики, которая показывает, какой объем кода протестирован;
* **Java** - универсальный язык программировани. Java имеет широкий спектр библиотек и платформ, доступных для тестирования, таких как JUnit, TestNG и Selenium
* **Git** - система контроля версий. С помощью Git-a можно откатить свой проект до более старой версии, сравнивать, анализировать или сливать свои изменения в репозиторий
* **Selenide** -  это фреймворк для автоматизированного тестирования веб-приложений на основе Selenium WebDriver. Его преимущества: изящный API, поддержка Ajax для стабильных тестов, мощные селекторы, простая конфигурация.
* **Библиотека Faker** - даёт возможность генерировать различные данные, подходящие для широкого спектра сценариев.
* **Библиотека Lombok** сокращает количество написанного кода, улучшая читаемость
* **Docker** - платформа для разработки и запуска контейнеров. Платформа позволяет быстрее тестировать приложения, запускать на одной машине требуемое количество контейнеров.
* **Allure** -  фреймворк, предназначенный для создания визуально наглядной картины выполнения тестов.
## Перечень и описание возможных рисков при автоматизации
